==========================================
ROLE SEPARATION NOTES
==========================================
- Super Admin can see "User Management" in sidebar (create/manage users).
  Super Admin CANNOT see business data (no opportunities, proposals, projects).

- Director is a business role. Director CAN approve proposals, gates, contracts,
  view margin/cost data, and access all project/sales data.
  Director CANNOT access User Management.

- Design Lead is the head of the Design department. Acts as technical authority
  for design reviews, DBD approvals, and drawing sign-offs.
  Senior to Design Engineer in the org hierarchy.

- PMO Manager is a portfolio governance role. Can view all projects, gates,
  milestones, issues, risks, and blockers across the portfolio. Can approve/reject
  gates (acting as the governance authority). Cannot access commercial data
  (contracts, invoices, margins, costs, financial close-out records).

- Project Engineer is an execution coordinator under the Project Manager.
  Can view assigned projects, prepare gate packages and submit (not approve) gates,
  manage milestones, log issues and risks, upload/revise documents.
  Cannot see CAPEX, margin, budget baseline, cost data, or contracts.
  All access is scoped to projects they are explicitly added to as a member.

- To create real users for the team, log in as see@pekatgroup.com and go to
  User Management → New User.

KNOWN ISSUES / NOTES
==========================================
- All test user display names end with "(Test)" — this is by design to
  distinguish seed accounts from real team members in the UI.
- The display name shown in the topbar and sidebar reflects the logged-in
  user's actual name from the database (fetched fresh on each login).
- Switching accounts: always use Sign Out first. The session cache is cleared
  on logout and login to ensure the correct user is displayed.
- Project Engineer sees no data until explicitly added as a ProjectMember
  on a project. This is by design (assigned scope).
