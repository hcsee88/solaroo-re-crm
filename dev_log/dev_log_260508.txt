## 2026-05-08 — CRM Sales Pipeline Review

### Review summary

- Reviewed current CRM progress and lifecycle coverage end-to-end.
- Confirmed the system is already strong on:
  - Auth / roles / permissions
  - Accounts / contacts / sites
  - Opportunities (incl. stage history, design-engineer assignment)
  - Proposals (versions, frozen assumptions, approval chain)
  - Contracts (incl. handover workflow + milestones + invoices)
  - Projects + PMO gates (G1–G6)
  - Documents (paginated list, upload, revision history, approve/reject)
  - Notifications (in-app bell, daily digest worker)
  - Audit log (global AuditService + auto-capture interceptor + EditMetaPill)
  - PMO dashboard (cash position, recent audit feed, RAG distribution)
  - Reports (/reports landing page; sales pipeline reporting endpoint)
- Identified the CRM **sales monitoring layer** as the next-best improvement
  area. Management lacks daily visibility into:
    - overdue follow-ups
    - stale opportunities
    - opportunities without a next action
    - proposal follow-up status
    - salesperson activity volume
    - expected-close discipline
    - pipeline forecast & overall sales pipeline health
- Decided to prioritise sales pipeline discipline before O&M.

### Product decision

- Sales activity tracking will be **manual only** in V1.
- No email, WhatsApp, call, Outlook, Gmail, calendar, telephony, or AI
  integration in this phase. Salespeople perform actions outside the CRM and
  manually log them.
- The CRM tracks activity notes and next-action due dates for visibility.
- The discipline tool is the **next-action due date** — every open
  opportunity should have one and only one open next action.

### Recommended next module

**Sales Pipeline Monitoring V1**, build sequence:

1. Manual activity logging (call / email / WhatsApp / meeting / site visit /
   proposal follow-up / general note) — single Activity table, no integrations.
2. Next-action tracking — type, owner, due date, status; one open action
   per live opportunity.
3. Opportunity health indicators — Healthy / At Risk / Stale / Overdue;
   computed at read time, never stored.
4. Opportunity list filters + saved views — My Open, Closing this month,
   Closing this quarter, No next action, Overdue, No activity 14d / 30d,
   Proposal submitted, High value, Won this month, Lost this month.
5. Sales pipeline dashboard at `/sales-pipeline` — pipeline summary, by-stage
   breakdown, sales activity volume, follow-up monitoring, proposal monitoring,
   closing forecast, top 10, won this month.
6. Internal notification hook points (in-app only) — overdue next action
   (daily), proposal awaiting follow-up (3d silence), stale opportunity
   (14d silence). Routed via existing Notification table + worker.

### Rationale

- Management needs better visibility into sales follow-up discipline.
- Opportunities should not sit idle without a next action.
- Proposal follow-up needs to be visible.
- Lightweight manual logging gives immediate value without integration
  complexity. We can layer email/WhatsApp/calendar later if the manual
  rhythm proves the workflow has product-market fit.

### Deferred

- O&M module (item 10 of original MVP build order).
- Email notification delivery (BullMQ worker still uses setInterval for
  daily digest only).
- WhatsApp / call / calendar / telephony integration.
- AI automation (auto-summary, auto-next-action suggestions).
- Advanced comments / threading on activities.
- Attachments on activities (use Documents module).

## 2026-05-08 (later same day) — Sales Pipeline Lite

### Why the dial-back

After shipping commit 33ea21e (Sales Pipeline V1 alignment with 7d AT_RISK /
14d STALE thresholds + 3 spec docs) the team reviewed the result and
concluded the dashboard was too dense and the thresholds too pressuring for
a 15-person engineering business. The CRM should feel like a helpful
operating system, not a stressful reporting tool.
