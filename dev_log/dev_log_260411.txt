A. SELF-SERVICE PASSWORD CHANGE
────────────────────────────────────────────────────────────────────────────────

Feature: Logged-in users can change their own password from the topbar menu.

Files changed:
  apps/api/src/modules/auth/auth.service.ts
    - Added changePassword(userId, currentPassword, newPassword) method
    - Verifies current password with bcrypt.compare()
    - Validates new password >= 8 characters
    - Hashes new password with bcrypt.hash(pwd, 12) and persists

  apps/api/src/modules/auth/auth.controller.ts
    - Added PATCH /auth/change-password endpoint
    - Uses @CurrentUser() decorator — requires valid JWT session

  apps/web/src/components/layout/topbar.tsx
    - Added ChangePasswordModal component (inline, self-contained)
    - Fields: currentPassword, newPassword, confirmPassword
    - Client-side validation: min 8 chars, passwords must match
    - Shows 1.5s success state then auto-closes
    - Accessible from the user dropdown → "Change Password" menu item
    - Uses KeyRound icon from lucide-react

B. PMO INTEGRATION — V1
────────────────────────────────────────────────────────────────────────────────

Approach: Thin governance layer inside the CRM. Not a separate system.
PMO_MANAGER gets cross-portfolio view + gate approval rights. No finance access.

── B1. Schema changes (packages/db/prisma/schema/projects.prisma) ──────────────

  Project model — added:
    pmoStatusNote     String?      PMO governance assessment note
    lastPmoReviewAt   DateTime?    Last formal PMO review timestamp

  ProjectGate model — added:
    pmoFlagged    Boolean  @default(false)   Flagged for PMO attention
    pmoComment    String?                    PMO review comment
    pmoReviewAt   DateTime?                  Last PMO gate review timestamp

  Milestone model — added:
    baselineDate  DateTime?   Frozen original committed date (never changed)
                              Enables schedule drift detection vs targetDate
