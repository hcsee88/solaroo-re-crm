─── ARCHITECTURE NOTES ─────────────────────────────────────────────────────────

Document scope enforcement:
  assigned scope → documents where:
    - ownerUserId = user.id  OR
    - projectId links to a project where user is PM or ProjectMember  OR
    - opportunityId links to an opportunity where user is owner or OpportunityMember

  all/team scope → no filter (DIRECTOR, PMO, SALES_MANAGER, DESIGN_LEAD, FINANCE_ADMIN)

Download mechanism:
  Direct window.open() to /api/documents/:id/download
  Browser sends httpOnly cookie automatically → auth passes → file streams
  Content-Disposition: attachment; filename="DocTitle_RevA.pdf"

Upload file storage paths:
  Opportunity-linked: uploads/opportunities/{opportunityId}/{categorySlug}/{timestamp}-{name}
  Project-linked:     uploads/projects/{projectId}/{categorySlug}/{timestamp}-{name}

  Re-upload / add new revision UI
  Transmittal management UI
  Bulk document actions
  Document-to-gate-deliverable linking UI
  Full-text document content search

Session continuation: PROJECT_ENGINEER Role Integration
================================================================================

TASK
────
Introduce the PROJECT_ENGINEER role across seed, permissions, UI nav, and docs.
This role did not previously exist in the system.

BUSINESS RATIONALE
──────────────────
A Project Manager oversees multiple projects and holds approval authority.
A Project Engineer is the on-the-ground execution coordinator — they track
deliverables, upload documents, log issues/risks, and prepare gate packages
for PM review. They have no commercial or approval authority.
